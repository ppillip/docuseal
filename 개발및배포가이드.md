# 포시즌에이아이 의 DocuSeal 개발 및 배포 가이드

이 문서는 DocuSeal 프로젝트의 개발 환경 설정부터 운영 서버 배포까지의 모든 과정을 담고 있는 통합 가이드입니다.

---

## 1. 개발 환경 (Development Mode)

코드를 수정하고 즉시 결과를 확인해야 하는 개발 단계에서 사용합니다.

### 🚀 구동 방법

```bash
./run.sh
```

### 💡 주요 특징

- **실시간 반영**: `app/controllers`, `app/views`, `config/locales` 등의 소스 코드를 수정하면 별도의 재시작 없이 즉시 반영됩니다.
- **접속 주소**: [http://localhost:9091](http://localhost:9091)
- **데이터 보관**: 프로젝트 루트의 `docuseal-data` 폴더에 데이터베이스와 설정이 저장됩니다.
- **간편함**: 복잡한 빌드 과정 없이 공식 Docker 이미지를 기반으로 코드만 얹어서 실행합니다.

---

## 2. 운영 환경 (Production Mode)

실제 서비스를 외부 사용자에게 오픈할 때 사용하는 무결점(Zero-Failure) 배포 방식입니다.

### 🛠️ 배포 순서

#### 1단계: 운영용 이미지 빌드

박사님의 최신 소스 코드를 이미지 내부로 완전히 결합(Bake)합니다.

```bash
./build_prod.sh
```

#### 2단계: 운영 서버 실행

빌드된 이미지를 기반으로 PostgreSQL과 함께 안전하게 구동합니다.

```bash
# 기본값 (localhost:9090)으로 실행 시
./run_prod.sh

# 고정 IP와 특정 포트(예: 9090)를 지정하여 실행 시
HOST=123.123.123.123 PORT=9091 ./run_prod.sh
```

- **IP 전용 환경**: 도메인이 없는 경우 `HOST`에 고정 IP를 입력하면 Caddy가 해당 IP로 요청을 받아줍니다. (단, IP 직접 접속 시 HTTPS는 지원되지 않습니다.)
- **포트 변경**: 다른 포트가 필요한 경우 `PORT` 변수를 통해 자유롭게 변경 가능합니다.

---

### 🛡️ 운영 환경의 강점

- **절대적 안정성**: 외부 코드 파일에 의존하지 않고 이미지 내부의 코드로만 동작하여 파일 유실 위험이 없습니다.
- **고성능 모드**: Rails의 Production 모드로 동작하며, 정적 파일 서빙 최적화가 적용됩니다.
- **강력한 DB**: SQLite 대신 운영용 **PostgreSQL 15**를 사용하여 대량의 데이터를 안전하게 관리합니다.
- **보안 및 도메인**: Caddy를 통해 HTTPS(SSL) 자동 갱신 및 80/443 표준 포트 서비스를 지원합니다.
- **접속 주소**: [http://localhost](http://localhost) (또는 설정된 도메인)

---

## 3. 관리 및 트러블슈팅

### 주요 파일 설명

- `run.sh`: 개발용 통합 실행 스크립트
- `build_prod.sh`: 운영용 이미지 생성 도구
- `run_prod.sh`: 운영 서버 배포 스크립트
- `docker-compose.prod.yml`: 운영 전용 서비스 구성도

### Docker 관련 체크

- **Docker 실행 확인**: 모든 스크립트는 Docker Desktop이 실행 중이어야 동작합니다.
- **포트 충돌**: 개발 모드는 9090 포트, 운영 모드는 80/443 포트를 사용하므로 다른 프로그램과 충돌이 없는지 확인해 주세요.
